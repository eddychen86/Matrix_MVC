輕量級社群平台

1. 專案概觀

1.1 專案簡介

本專案旨在開發一個輕量級的社群平台，專注於提供核心的社群互動體驗。使用者可以註冊帳號、發表個人動態、對他人的貼文進行留言與按讚，形成一個純粹、友善的線上交流空間。專案目標是打造一個穩定、易用且專注於內容分享與互動的應用程式。

1.2 專案目標

在為期 1.5 個月（六週） 的時間内，由 四人 開發團隊合作完成此專案的最小可行性產品 (Minimum Viable Product, MVP)，實現所有核心社群功能，並準備好進行最終的成果展示。

2. 專案範疇

✓ 範圍內功能 (In-Scope)

  • 使用者系統: 基於 Email/密碼的註冊、登入與個人資料管理。
  • 社群核心: 建立與閱讀文字貼文。
  • 核心互動: 對貼文進行「按讚」與「留言」。

✗ 範圍外功能 (Out-of-Scope)

  • Web3 整合功能: 包含 NFT 展示、ENS 域名顯示等。
  • 即時通訊: 一對一或群組聊天功能。
  • 任何形式的金流支付系統。
  • 第三方開發者插件系統。
  • 上傳圖片或影片至貼文。

4. 功能性需求

  • 註冊: 使用者可使用 Email 和密碼建立新帳號。密碼需加密儲存。
  • 登入/登出: 使用者可登入以存取完整功能，並可隨時登出。
  • 查看個人檔案: 任何人皆可查看使用者的個人檔案頁面，顯示其基本資料與發布過的貼文。
  • 編輯個人檔案: 使用者登入後，可編輯自己的顯示名稱等基本資訊。

  • 建立貼文: 登入使用者可發布純文字內容的貼文。
  • 查看時間軸: 首頁將以時間倒序顯示所有公開的貼文。
  • 新增留言: 登入使用者可在任一貼文下方新增文字留言。
  • 按讚貼文: 登入使用者可對貼文按讚或取消讚。貼文將顯示總按讚數。
  • 刪除貼文/留言: 使用者可以刪除自己發布的貼文或留言。

5. 資料庫綱要

此為本專案的核心資料庫結構示意圖，展示了各個資料表之間的關聯。

Users (AspNetUsers)
  • Id (PK)
  • Email
  • PasswordHash
  • ... (Identity 預設欄位)
Posts
  • Id (PK)
  • Content (NVARCHAR)
  • CreatedAt (DATETIME)
  • AuthorId (FK to Users.Id)
Comments
  • Id (PK)
  • Content (NVARCHAR)
  • CreatedAt (DATETIME)
  • PostId (FK to Posts.Id)
  • AuthorId (FK to Users.Id)
Likes
  • Id (PK)
  • PostId (FK to Posts.Id)
  • UserId (FK to Users.Id)

8. 團隊分工

為了確保開發效率與權責分明，團隊將分為後端與前端兩組，各自負責明確的專業領域。

⚙️ 後端團隊 (2 人)

  • 資料庫設計: 規劃實體關係圖 (ERD) 與資料表綱要。
  • 資料庫建置: 使用 EF Core 進行資料庫遷移 (Migration) 與模型建立。
  • 後端 API 設計: 設計並實現標準化、易於使用的 RESTful API 接口 (用於留言、按讚等)。
  • 核心業務邏輯: 編寫所有 Controller 的商業邏輯。

🎨 前端團隊 (2 人)

  • UI/UX 設計: 負責網站整體的視覺風格、使用者流程與互動體驗設計。
  • 前端切版與樣式: 將設計稿轉換為響應式的 HTML 與 CSS (Bootstrap)。
  • 前端互動開發: 使用 Vue.js 實現非同步留言與按讚功能。
  • API 串接與狀態管理: 呼叫後端 API，並管理前端頁面的資料狀態。
